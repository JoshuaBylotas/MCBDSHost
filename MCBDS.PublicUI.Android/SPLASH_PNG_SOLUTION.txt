=== Custom PNG Splash Screen - Proper Setup Guide ===

THE ISSUE:
PNG files require different handling than SVG files for MAUI splash screens.
The error "drawable/mcbds_splash not found" means the PNG wasn't compiled properly.

SOLUTION OPTIONS:

=== OPTION 1: Convert PNG to SVG (RECOMMENDED) ===

Android MAUI splash screens work best with SVG files. Convert your PNG to SVG:

Tools to convert PNG to SVG:
- Online: https://convertio.co/png-svg/
- Photoshop: Export As ? SVG
- Inkscape: Import PNG, Trace Bitmap, Save as SVG
- GIMP with plugin

Steps:
1. Convert MCBDS Splash.png to mcbds_splash.svg
2. Place in: MCBDS.PublicUI.Android\Resources\Splash\mcbds_splash.svg
3. Update .csproj:
   <MauiSplashScreen Include="Resources\Splash\mcbds_splash.svg" Color="#FFFFFF" BaseSize="456,456" />

=== OPTION 2: Use PNG as MauiImage (Alternative) ===

If you must use PNG, add it as an image instead:

1. Keep: Resources\Splash\mcbds_splash.png
2. Update .csproj:

<ItemGroup>
    <!-- Keep default splash -->
    <MauiSplashScreen Include="Resources\Splash\splash.svg" Color="#FFFFFF" BaseSize="128,128" />
    
    <!-- Add your logo as an image -->
    <MauiImage Include="Resources\Splash\mcbds_splash.png" />
</ItemGroup>

Then modify the default splash.svg to reference your PNG as a background image.

=== OPTION 3: Create Simple SVG Wrapper ===

Create a simple SVG that embeds your PNG:

File: Resources\Splash\mcbds_splash.svg

<?xml version="1.0" encoding="UTF-8"?>
<svg width="456" height="456" version="1.1" xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink">
    <image x="0" y="0" width="456" height="456" xlink:href="mcbds_splash.png"/>
</svg>

Then keep both files:
- Resources\Splash\mcbds_splash.svg (wrapper)
- Resources\Splash\mcbds_splash.png (actual image)

Update .csproj:
<MauiSplashScreen Include="Resources\Splash\mcbds_splash.svg" Color="#FFFFFF" BaseSize="456,456" />

=== OPTION 4: Use Default Splash.svg and Replace Content ===

Easiest option - just edit the existing splash.svg file:

1. Open: Resources\Splash\splash.svg
2. Replace its content with your logo design
3. Keep the name as splash.svg
4. No .csproj changes needed!

=== WHY PNG DOESN'T WORK DIRECTLY ===

MAUI's MauiSplashScreen expects vector formats (SVG) because:
- SVG scales perfectly to any screen size
- PNG requires multiple resolutions (ldpi, mdpi, hdpi, xhdpi, etc.)
- Android's resource compiler (aapt2) expects specific naming
- MAUI Resizetizer generates intermediate drawables for SVG automatically

For PNG, you'd need to provide multiple resolutions manually, which MAUI doesn't do automatically for splash screens.
